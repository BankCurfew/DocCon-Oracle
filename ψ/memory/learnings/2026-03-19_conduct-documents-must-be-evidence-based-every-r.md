# Conduct documents must be evidence-based — every rule, violation, and score must

Conduct documents must be evidence-based — every rule, violation, and score must point to a specific source (thread #, file:line, test score, data analysis). Opinion-based quality standards get pushed back and lose authority. "Patterns Over Intentions" — DocCon Principle #2. Example: Jarvis Conduct sourced from Researcher (10,956 msgs), QA (50% PASS Round 1), AIA (domain review scores 3/10 accuracy). FA Tools Code Conduct scored 5.4/10 with 40+ violations each citing specific files and tsconfig settings.
