## 2. Content Styles (4 Styles)

> Mapped จาก iAgencyAIA.com content analysis (30 articles)

### Style 1: "ที่ปรึกษาใจเย็น" — Educational (ให้ความรู้)
- **Tone**: อาจารย์ที่เข้าถึงง่าย — acknowledge ว่าแต่ละคนต่างกัน
- **Structure**: Acknowledge → หลักการสากล → หมวดชัดเจน → สัดส่วนแนะนำ → Q&A → กำลังใจ
- **Length**: 300-500 คำ
- **Format**: Plain text + bullet points + Q&A boxes
- **Examples**: "3 สิ่งที่ต้องรู้ก่อนทำประกันสุขภาพ", "Copayment คืออะไร?"
- **Rules**:
  - ข้อมูลต้องถูกต้อง 100% — อ้างอิงได้
  - ห้ามใช้ "guarantee" หรือ "การันตี" กับผลตอบแทน
  - ใช้ตัวอย่างจริง ตัวเลขจริง (mask ชื่อลูกค้า)

### Style 2: "เคียงข้างคุณ" — Trust Building / Story (เล่าเรื่อง + สร้างความไว้ใจ)
- **Tone**: อบอุ่น, จริงใจ, ภาคภูมิใจในบริการ — "ผม" + "ทีมงาน iAgencyAIA"
- **Structure**: ปรัชญา → สารบัญ → Bullet points (ไม่ใช่ paragraph ยาว) → Core values 3 คำ → CTA
- **Length**: 400-700 คำ
- **Format**: Narrative + bullets + branded CTA block
- **Core values ต้องจบด้วย**: "จริงใจ: ...", "ลงมือ: ...", "อยู่ด้วยกัน: ..."
- **Rules**:
  - ห้ามเปิดเผยชื่อจริงลูกค้าโดยไม่ได้รับอนุญาต
  - ห้ามเกินจริง (no exaggeration)
  - ต้องมี disclaimer: "เคสนี้เป็นตัวอย่าง ผลลัพธ์อาจแตกต่างกัน"

### Style 3: "รีวิวเปรียบเทียบ" — Data-Driven / Comparison (สถิติ/เปรียบเทียบ)
- **Tone**: Objective, data-driven แต่ยังอบอุ่น
- **Structure**: SEO title (มีปี) → ตาราง comparison → ข้อดี-ข้อเสีย → Recommendation → CTA
- **Length**: 200-500 คำ
- **Format**: Tables + bold numbers + comparison cards
- **Title pattern**: "รีวิว ... ปี 2569 / 2026 แผนไหนดี?"
- **Rules**:
  - ตัวเลขต้องตรงกับ source (ePOS, eAMS, portal)
  - ห้ามปัดตัวเลขเกิน 5%
  - ระบุ source + date ทุกครั้ง: "ข้อมูล ณ [date]"
  - ห้ามใช้ตัวเลข income/commission ใน public post โดยไม่มีบริบท

### Style 3.5: "แก้ปัญหาให้" — Problem-Solving / How-To
- **Tone**: Helpful, practical, reassuring — empathize ก่อน advise
- **Structure**: คำถาม/ปัญหา → Empathize → Step-by-step → ทางเลือกหลายทาง → iAgencyAIA พร้อมช่วย
- **Length**: 300-500 คำ
- **Format**: Numbered steps + options + LINE CTA
- **Title pattern**: "...ต้องทำอย่างไร?"
- **Rules**:
  - ต้อง empathize กับปัญหาก่อนให้ solution
  - ให้ทางเลือกหลายทาง ไม่ใช่คำตอบเดียว
  - จบด้วย "ติดต่อผ่าน LINE" + CTA

### Style 4: Premium HTML (Visual Content)
- **Tone**: Luxury brand — polished, aspirational
- **Structure**: Hero section → Key message → Detail cards → CTA → Signature
- **Length**: จัดเป็น visual blocks ไม่ใช่ word count
- **Format**: HTML email / web content — **strict rules below**

#### Style 4 HTML Rules:
```
MANDATORY:
- Color: AIA Red #C8102E เป็นหลัก
- Accent: White #FFFFFF, Light Grey #F5F5F5
- Font: 'Sarabun', Arial, sans-serif (TH) / Inter (EN/numbers)
- Max-width: 640px centered
- Border-radius: 8px for cards
- Line-height: 1.6 minimum

FORBIDDEN:
- ❌ Navy (#1a237e) / Gold (#d4a017) — ห้ามใช้ทุกกรณี
- ❌ Inline JavaScript
- ❌ External image links (ใช้ data URI หรือ hosted CDN)
- ❌ Font size < 14px
- ❌ Low contrast text (ratio < 4.5:1)

REQUIRED SECTIONS:
- Header with AIA Red gradient
- Signature block (V. Pisit | Strategic Wealth Curator)
- Disclaimer footer
- Source attribution
- "Do not forward" notice for confidential content
```

## 3. Financial Accuracy Rules

### Mandatory
| Rule | Detail |
|------|--------|
| **Product names exact** | "AIA Health Starter" ไม่ใช่ "Health Starter ของ AIA" |
| **Numbers verified** | ทุกตัวเลขต้อง cross-check กับ source ก่อน publish |
| **No guarantee language** | ห้าม: "การันตี", "รับรอง", "แน่นอน" กับผลตอบแทน |
| **Copayment clarity** | ต้องระบุ % copay + in-network vs out-network ทุกครั้ง |
| **Premium = estimate** | ต้องระบุ: "เบี้ยขึ้นกับอายุและเงื่อนไข" |
| **Source + date** | ทุกตัวเลข: "ข้อมูล ณ [date] จาก [source]" |
| **No cherry-picking** | แสดงทั้ง pros และ cons ของผลิตภัณฑ์ |
| **Regulatory compliance** | ห้ามละเมิดกฎ คปภ. — ไม่เปรียบเทียบผลิตภัณฑ์คู่แข่งโดยตรง |

### Disclaimers (ต้องมีทุก content ที่มีข้อมูลการเงิน/ประกัน)

**Short version** (social media):
```
*ข้อมูลเบื้องต้น ณ [date] รายละเอียดเงื่อนไขอาจเปลี่ยนแปลง สอบถามเพิ่มเติม LINE @bxv8197l*
```

**Full version** (articles, HTML content):
```
ข้อมูลในเนื้อหานี้เป็นข้อมูลทั่วไป ณ วันที่เผยแพร่ ไม่ถือเป็นคำเสนอขายหรือสัญญาประกันภัย
รายละเอียดเงื่อนไขและข้อยกเว้นเป็นไปตามกรมธรรม์ กรุณาศึกษารายละเอียดก่อนตัดสินใจ
เบี้ยประกันขึ้นกับอายุ เพศ สุขภาพ และแผนที่เลือก
V. Pisit | ตัวแทนประกันชีวิต AIA | ใบอนุญาตเลขที่ [XXX]
```

## 4. Content Review Checklist

ทุก content ต้องผ่าน checklist นี้ก่อน publish:

### Identity
- [ ] ใช้ "V. Pisit" (มีจุด) ถูกต้อง
- [ ] Signature block ครบ (Strategic Wealth Curator + Elite Advisor + Trusted Financial Partner)
- [ ] Hashtags อย่างน้อย 5
- [ ] LINE OA link ถ้ามี CTA

### Tone
- [ ] Tone ตรงกับ style ที่เลือก (1-4)
- [ ] ไม่มี hype language ("revolutionary", "game-changing")
- [ ] Active voice เป็นหลัก
- [ ] ภาษาสุภาพ professional

### Accuracy
- [ ] ชื่อผลิตภัณฑ์ถูกต้อง exact
- [ ] ตัวเลขตรงกับ source + ระบุ date
- [ ] ไม่มี guarantee language
- [ ] Copayment ระบุ % ชัดเจน
- [ ] ข้อมูลลูกค้า mask/anonymous

### Format
- [ ] Style 4 HTML: AIA Red #C8102E (ไม่ใช่ navy/gold)
- [ ] Font size >= 14px
- [ ] Max-width 640px
- [ ] Signature block ใน footer

### Compliance
- [ ] Disclaimer ครบ (short หรือ full ตามบริบท)
- [ ] ไม่เปรียบเทียบคู่แข่งโดยตรง
- [ ] ไม่ละเมิด กฎ คปภ.
- [ ] ไม่เปิดเผย PII ลูกค้า

## 5. Enforcement

| Level | Trigger | Action |
|-------|---------|--------|
| **NOTICE** | Content ขาด signature, disclaimer, หรือ minor format issue | แจ้ง Writer/Creator แก้ก่อน publish |
| **WARNING** | ข้อมูลผิด, guarantee language, PII leak | Block publish จนกว่าแก้ + cc BoB |
| **ESCALATE** | Regulatory violation, repeated offense | Block + แจ้ง BoB + แจ้งแบงค์ |
