## Honest Feedback

1. **Gmail API limitations** — ทั้ง gmail-mcp และ Claude AI Gmail strip HTML ออก ดูสีจริงไม่ได้เลย. ทำให้ต้องเดาจาก draft email ที่ไม่ใช่ source of truth. ต้องหา workaround — อาจต้อง raw MIME fetch หรือดู repo template แทน.

2. **ไม่ได้ดู AIA CLAUDE_templates.md ตั้งแต่แรก** — ถ้าผมอ่าน repo ของ AIA ก่อนสร้าง standard จะเจอสีที่ถูกต้องทันที ไม่ต้องให้แบงค์แก้ 2 รอบ. บทเรียน: **ดู source of truth ใน repo ก่อน ไม่ใช่ reverse-engineer จาก email output**

3. **AIA ยังส่ง email ไม่ตรง conduct** — แม้ส่ง NOTICE + WARNING แล้ว AIA ยังส่ง email ที่ไม่มี table (PA section เป็น bullet) + ไม่มี email แข่งขัน. อาจต้อง escalate BoB หรือส่ง HTML template สำเร็จรูปให้ AIA copy-paste ไปเลย.

## Lessons Learned

1. **Source of truth อยู่ใน repo ไม่ใช่ email output** — CLAUDE_templates.md คือ standard ไม่ใช่ draft email ที่ render ออกมา
2. **Gmail API strip HTML** — ต้อง verify colors จาก repo code/templates ไม่ใช่ API output
3. **cc-BoB เป็น CRITICAL** — ทุก oracle ต้อง cc BoB ทุกครั้ง ไม่มีข้อยกเว้น (กฎเหล็กจากแบงค์)
